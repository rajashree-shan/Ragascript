ðŸŽ¶ Swara System (Variables & Values)

Variables in RagaScript are called swaras, representing musical notes that carry meaning and emotion. Each swara declaration includes its raga type and initial value:

// Declaring swaras (variables) in different ragas kalyani<number> sa = 1.618; // Golden ratio in Kalyani's mathematical purity mohanam<string> ri = "melody"; // Simple string in Mohanam's elegance behag<array> ga = [Sa, Ri, Ga, Pa, Dha]; // Dynamic array in Behag's flow nalinakanthi<object> ma = { tala: "Adi", gati: "Chatusra", tempo: "Madhyama" };

Notice how each swara carries not just data, but the emotional and structural context of its raga. A number in Kalyani suggests mathematical relationships, while a string in Mohanam implies melodic simplicity.

Alapana Blocks (Dynamic Code Exploration)

The heart of RagaScript lies in alapana blocks - sections where structured improvisation occurs. Like musical alapana, these blocks start with basic elements and develop organically:

alapana in Kalyani { // Establish the foundation (Arohana/Avarohana) establish Sa, Ri2, Ga3, Ma2, Pa, Dha2, Ni3; // Explore lower octave (Mandra sthayi) explore_mandra { if (mood == "contemplative") { dwell_on Sa, Pa; // Stay on stable notes gentle_gamaka Ri2 -> Ga3; } } // Develop middle octave (Madhya sthayi) explore_madhya { phrase1: Sa Ri2 Ga3 Ma2 Pa; phrase2: Pa Dha2 Ni3 Sa_upper; connect phrase1 -> phrase2 with smooth_transition; } // Soar to upper octave (Tara sthayi) explore_tara { climactic_phrase: Dha2 Ni3 Sa Ma2 Ga3; return_gracefully to Sa; } }

Alapana blocks encourage exploration within structure - just like musical alapana explores a raga's personality while respecting its rules. The code adapts and evolves based on conditions, creating beautiful, organic algorithms.
